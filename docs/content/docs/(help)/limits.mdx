---
title: Limits
description: Check request, connection, and startup limits before opening a tunnel.
---

These limits apply to gateway instances, local relay processes, and Vercel Queue traffic.

- The public URL works only while `tt http` or `tt dev` is running.
- HTTP requests time out after 30 seconds.
- Request and response bodies are limited to 4 MiB (4,194,304 bytes) each.
- Each gateway instance accepts up to 32 concurrent public WebSocket connections per tunnel slug.
- The local relay socket pool can contain 1 to 16 connections and defaults to 2.
- `tt dev` waits up to 60 seconds for the local server before it stops the managed process and tunnel.
- `tt list` waits up to five seconds for a response and accepts response bodies up to 1 MiB.
- Gateway tunnel listing stops after 10,000 presence events rather than returning a partial result.

Vercel may run several gateway instances. The WebSocket connection limit applies independently to each instance.
