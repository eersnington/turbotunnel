This page describes saved gateway settings, runtime overrides, and the production variables managed by `tt deploy`.

## Saved configuration

`tt deploy` writes `~/.turbotunnel/config.json` after it verifies the gateway:

```json
{
  "project": "ttabc123-turbotunnel",
  "slug": "ttabc123",
  "relayDomain": "{slug}-turbotunnel.vercel.app",
  "queueRegion": "iad1",
  "relaySecret": "ttsec_example_redacted"
}
```

The relay secret authenticates local relay connections and requests to the tunnel-list endpoint. It does not protect traffic sent to a public tunnel URL. Do not commit or share this file.

`tt deploy` also keeps generated deployment files in `~/.turbotunnel/relay`. It preserves that directory's `.vercel` project metadata between deployments.

## Environment variables

Use environment variables for per-process overrides and local gateway development. `tt http` and `tt dev` read these variables:

| Variable                   | Purpose                                      |
| -------------------------- | -------------------------------------------- |
| `TURBOTUNNEL_SLUG`         | Public slug for the current tunnel.          |
| `TURBOTUNNEL_BASE_DOMAIN`  | Base domain or `{slug}` host pattern.        |
| `TURBOTUNNEL_RELAY_DOMAIN` | Older alias for `TURBOTUNNEL_BASE_DOMAIN`.   |
| `TURBOTUNNEL_RELAY_SECRET` | Gateway bearer secret.                       |
| `TURBOTUNNEL_RELAY_URL`    | Explicit relay origin for local development. |

`tt http` resolves configuration in this order:

1. Command-line options
2. Environment variables
3. Saved configuration
4. Generated defaults

`tt dev` has no tunnel configuration options, so it starts with environment variables and then uses saved configuration and generated defaults.

For the domain, `TURBOTUNNEL_BASE_DOMAIN` takes precedence over `TURBOTUNNEL_RELAY_DOMAIN`.

## Vercel environment variables

`tt deploy` sets these variables in the project's Production environment:

| Variable                   | Value                                       |
| -------------------------- | ------------------------------------------- |
| `TURBOTUNNEL_BASE_DOMAIN`  | The configured base domain or host pattern. |
| `TURBOTUNNEL_RELAY_SECRET` | The generated or reused relay secret.       |
| `TURBOTUNNEL_QUEUE_REGION` | The configured queue region.                |

## Gateway development variables

When you run the gateway outside Vercel, it also reads these variables:

| Variable | Purpose | Development default |
| --- | --- | --- |
| `TURBOTUNNEL_BROKER` | Select `memory` or `vercel` | `memory` when `NODE_ENV=development` |
| `NODE_ENV` | Select development behavior | None |
| `PORT` | Gateway HTTP port | `3002` |

The local gateway defaults to domain `localhost`, relay secret `dev_secret`, and Queue region `iad1`. Do not use the development secret for a public deployment.
