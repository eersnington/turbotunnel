---
title: tt dev
description: Run a project dev server and public tunnel as one managed process.
---

```sh
tt dev [project] [--port <port>] [access options]
tt dev [project] [--port <port>] -- <command> [args...]
```

`tt dev` runs a configured project and its public tunnel in one lifecycle. It finds the nearest `turbotunnel.json` and selects a project by name or current directory. It waits for HTTP readiness and stops both processes when you press `Ctrl-C`. Without project configuration, it runs the nearest package's `dev` script.

```sh
tt dev dashboard
tt dev dashboard --password
tt dev dashboard --allow-ip 203.0.113.0/24
```

## Select a port

Pass `--port` to select the local target. Otherwise, Turbotunnel uses a port from the custom command or allocates a free port.

| Option          | Description                                    | Default                                    |
| --------------- | ---------------------------------------------- | ------------------------------------------ |
| `--port <port>` | Select the local server port, from 1 to 65535. | Custom command port, otherwise a free port |

## Command arguments

The second form replaces the package `dev` script with an explicit executable and arguments:

| Input       | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `<command>` | Executable to start directly, without a shell.               |
| `[args...]` | Child arguments; supported port options may be normalized.   |
| `--`        | Separator between Turbotunnel options and the child command. |

Turbotunnel recognizes Next.js, Vite, Astro, Nuxt, and Storybook commands. It replaces an existing framework port argument or appends the selected port in the framework's normal form. Vite also receives `--strictPort`.

For other executables, `--port <port>` or `--port=<port>` in the child arguments selects the tunnel port. Turbotunnel normalizes either form to `--port <port>`.

You can also pass `--port <port>` before the separator. For an unrecognized executable, Turbotunnel does not append a child argument.

Turbotunnel waits up to 60 seconds for the selected port to accept an HTTP connection. If the deadline expires, it stops the child process and tunnel.

## Package scripts

Run the nearest package's `dev` script on a free port:

```sh
tt dev
```

Use a fixed port when the script already hardcodes one:

```sh
tt dev --port 1024
```

At a monorepo root, pass the configured project name. From a configured project root, `tt dev` selects that project automatically. Project roots may be `docs`, `packages/dashboard`, or another relative directory. Turbotunnel does not assume an `apps/*` layout or run a workspace-wide task unless you configure one.

## Custom commands

Everything after `--` belongs to the child process and runs without a shell:

```sh
tt dev -- vite --host 0.0.0.0
```

When running the repository CLI through Bun, preserve both argument boundaries:

```sh
bun run tt -- dev --port 1024 -- bun run --cwd docs dev
```

Turbotunnel prints the custom command before launch. It escapes control characters and redacts values for options containing `auth`, `credential`, `key`, `password`, `secret`, or `token`. It also redacts uppercase environment assignments with those terms in their names.

## Terminal output

The startup block shows the public URL, local target, relay count, command, and working directory. Turbotunnel marks where child output begins and tunnel lifecycle output resumes:

```text
. turbotunnel beta

  Public           https://demo-turbotunnel.vercel.app/
  Local            http://localhost:1024
  Relays           2 sockets
  Process          bun run --cwd docs dev
  Directory        /Users/you/project

Waiting for local app at localhost:1024
──── dev server ────────────────────────
$ blume dev --port 1024
…

──── turbotunnel ────────────────────────
✓ Local app ready
· Connecting 2 relay sockets
✓ Tunnel ready in 3.1s

  Press Ctrl-C to stop
```

Turbotunnel does not intercept or prefix child output. The child keeps its normal colors, formatting, prompts, and terminal behavior.
