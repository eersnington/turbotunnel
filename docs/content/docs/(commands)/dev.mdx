---
title: tt dev
description: Run a project dev server and public tunnel as one managed process.
---

```sh
tt dev [--port <port>]
tt dev [--port <port>] -- <command> [args...]
```

`tt dev` finds the nearest `package.json` and starts its `dev` script. Pass a command after `--` to run another executable. Turbotunnel waits for the local HTTP server and then opens the tunnel. Press `Ctrl-C` to stop both processes.

## Select a port

Pass `--port` to select the local target. Otherwise, Turbotunnel uses a port from the custom command or allocates a free port.

| Option          | Description                                    | Default                                    |
| --------------- | ---------------------------------------------- | ------------------------------------------ |
| `--port <port>` | Select the local server port, from 1 to 65535. | Custom command port, otherwise a free port |

## Command arguments

The second form replaces the package `dev` script with an explicit executable and arguments:

| Input       | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `<command>` | Executable to start directly, without a shell.               |
| `[args...]` | Child arguments; supported port options may be normalized.   |
| `--`        | Separator between Turbotunnel options and the child command. |

The child receives these environment variables:

| Variable           | Value                     |
| ------------------ | ------------------------- |
| `PORT`             | The selected local port.  |
| `TURBOTUNNEL_URL`  | The resolved public URL.  |
| `TURBOTUNNEL_HOST` | The public tunnel host.   |
| `TURBOTUNNEL_SLUG` | The resolved tunnel slug. |

Turbotunnel recognizes these framework commands: Next.js, Vite, Astro, Nuxt, and Storybook. It replaces an existing framework port argument and appends the selected port in the framework's normal form. Vite also receives `--strictPort`.

For other executables, `--port <port>` or `--port=<port>` in the child arguments selects the tunnel port. Turbotunnel normalizes either form to `--port <port>`. Without that argument, the executable must read `PORT`.

You can also pass `--port <port>` before the separator. For an unrecognized executable, this sets `PORT` but does not append a child argument.

Turbotunnel waits up to 60 seconds for the selected port to accept an HTTP connection. If the deadline expires, it stops the child process and tunnel.

## Package scripts

Run the nearest package's `dev` script on a free port:

```sh
tt dev
```

Use a fixed port when the script already hardcodes one:

```sh
tt dev --port 1024
```

At a monorepo root, the nearest package is the root package. If its `dev` script starts several workspaces, run Turbotunnel from the application package or pass a custom command instead.

## Custom commands

Everything after `--` belongs to the child process and runs without a shell:

```sh
tt dev -- vite --host 0.0.0.0
```

When running the repository CLI through Bun, preserve both argument boundaries:

```sh
bun run tt -- dev --port 1024 -- bun run --cwd docs dev
```

Turbotunnel prints the custom command before launch. It escapes control characters and redacts values attached to options containing `auth`, `credential`, `key`, `password`, `secret`, or `token`. It also redacts uppercase environment assignments whose names contain those terms.

## Terminal output

The startup block shows the public URL, local target, relay count, command, and working directory. Turbotunnel then marks where inherited child output begins and where tunnel lifecycle output resumes:

```text
. turbotunnel 0.0.2

  Public           https://demo-turbotunnel.vercel.app/
  Local            http://localhost:1024
  Relays           2 sockets
  Process          bun run --cwd docs dev
  Directory        /Users/you/project

Waiting for local app at localhost:1024
──── dev server ────────────────────────
$ blume dev --port 1024
…

──── turbotunnel ────────────────────────
✓ Local app ready
· Connecting 2 relay sockets
✓ Tunnel ready in 3.1s

  Press Ctrl-C to stop
```

Turbotunnel does not intercept or prefix child output. The child keeps its normal colors, formatting, prompts, and terminal behavior.
